## 0. 当前 task mode（direct-dev 默认，受控窗口任务包）

当前仓库普通内部开发默认按 `DIRECT_DEV_DEFAULT` 运行，不再把 task packet / scoped subpacket 作为普通开发前置门槛。触及高风险、对外/live、release gate、approval/audit 语义、schema/migration、跨阶段机器契约、大批量治理窗口，或人类明确要求走小包时，才进入 task packet / scoped subpacket。

| task mode | 含义 | 默认要求 |
|---|---|---|
| `direct_dev` | 普通代码修复、测试修复、文档小修、局部重构、非 live 内部功能实现 | 不要求先切换 `current_task`；按影响面定位、最小实现、验证、汇报或提交 |
| `activation` | 激活新 task packet / scoped subpacket，冻结目标、非目标、改动范围与 required scripts | 仅受控窗口必须先锁 `current_task` 语义，再进入执行 |
| `scoped_execution` | 按已声明 task packet 做本轮最小闭环执行 | 仅受控窗口限制在 declared / allowed 范围内改动；路线图只作导航 |
| `closeout` | 本轮执行完成后的脚本复跑、停机汇报与收口 | 不自动进入下一包，不自动改 readiness / external release 受控开放要求 |

补充说明：
- `current_task` 只锁受控窗口当前包，`product_task_library` 只选下一包；
- `source_blueprint_registry` 负责 source blueprint 合法性；`operator_assignment_roster_defaults` 负责 stage7/8/9 roster 稳定来源；
- `repo_status.md` 与 `milestone_status.yaml` 是状态真源；
- `docs/AX9S_当前主线导航图.md` 只负责导航，不决定当前执行顺序；
- 未显式说明 task mode 时，普通开发默认按 `direct_dev` 解释；受控窗口默认按 `scoped_execution` 解释。

## 1. change class 与动作等级定义

| change_class | 含义 | 对应自动化动作 |
|---|---|---|
| `LOW_RISK_DIRECT` | 低风险同步类变更，不触及 runtime / governance / release / automation gate 高风险域 | `DIRECT_EDIT` |
| `DRAFT_WITH_REVIEW` | 中风险 machine asset / scripts / tests 变更，可先出 draft/patch，但必须人工 review | `DRAFT_ONLY` |
| `MANDATORY_HUMAN_REVIEW` | 高风险改动，必须声明 human review 与 owner signoff | `MANUAL_APPROVAL_REQUIRED` |
| `STOP_AND_ESCALATE` | 必须立即停机转人工，不得继续自动推进 | `BLOCKED` |

| `shared_runtime_core` | `MANDATORY_HUMAN_REVIEW` | `architecture_owner` + `governance_owner` + `testing_owner` |

补充说明：

- change class 必须主要按触及文件 / 资产类别 / 影响面判定，不接受自由文本降级。
- 普通路线图片段可按 `DIRECT_DEV_DEFAULT` 推进；高风险/live/机器契约路线图片段必须先形成 task packet，再按 change class 进入对应门禁。

## 3. 动作门禁矩阵（冻结）

| 动作类型 | action_class | 说明 |
|---|---|---|
| 文档同步（D1/D2/D3/D6/D7/D8/D9/D10/D11/D12/D13/D14） | `DIRECT_EDIT` | 可按当前 AGENTS / 人类指令修正文或补表；不得引入第二套正式语义 |
| 新增/更新 control 资产（`control/*`） | `DIRECT_EDIT` | 普通同步可 direct edit；受控窗口必须同步 D1/AGENTS/当前任务 |
| 新增/更新 contracts/handoff/testing/governance 机器资产 | `DRAFT_ONLY` | 普通一致性同步可 direct-dev；改变正式契约、release、governance 或审批审计语义时必须复核 |
| 修改 scripts | `DIRECT_EDIT` | 普通脚本修复可 direct-dev；release/readiness/gate 脚本按 review gate 升级 |
| 修改 `src/*` 或 `tests/*` | `DIRECT_EDIT` | 普通内部代码/测试可 direct-dev；live、release、schema/migration、Stage 8/9 高风险执行按 review gate 升级 |
| 任何外部系统集成或外部发布 | `MANUAL_APPROVAL_REQUIRED` | external release 是 controlled opening；live 前必须 controlled-opening gate + release checklist + 审批链 + 审计链 + operator action |
| 真实触达 / 真实支付 / 真实交付 | `MANUAL_APPROVAL_REQUIRED` | 可作为受控开放目标能力推进；live 前必须 provider config、sandbox、审批、审计、operator action 和验收通过 |
| 高限制字段（自然人高限制、治理补证）自动外发 | `BLOCKED` | 必须人工审批链 |

补充说明：
- 动作级门禁不替代 change class；最终以 `review_gate_matrix -> action_matrix` 映射决定是否可 direct/draft/stop；
- `MANDATORY_HUMAN_REVIEW` 及以上等级不得继续当作 direct batch。

## 4. task packet 硬门槛（冻结）

| 必填字段 | 说明 |
|---|---|
| `objective` | 任务目标 |
| `non_goals` | 明确非目标 |
| `affected_stages` | 涉及阶段或跨阶段门禁面 |
| `declared_changed_paths` | 本批实际声明要改的文件 |
| `allowed_modification_paths` | 允许修改范围 |
| `forbidden_modification_paths` | 禁止修改范围 |
| `impacted_assets` | 受影响 docs / control / contracts / scripts / tests |
| `required_scripts` | 必跑脚本 |
| `stop_conditions` | 本批停机条件 |
| `definition_of_done` | 完成标准 |
| `deliverables` | 产物清单 |
| `risk_level` | 风险级别 |
| `change_class` | 变更分类 |
| `change_domains` | 命中的高风险域 |
| `human_review_required` | 是否必须 human review |
| `owner_reviews_required` | 是否必须 architecture / governance / automation / testing 等 owner review |

补充说明：
- 正式 machine-readable task packet 统一落在 `control/current_task.yaml -> currentTask.task_packet`；
- 缺上述关键字段时，`scripts/check-task-packet.ps1` 必须失败。
- controlled task packet / scoped subpacket 窗口真正改文件前建议先做一次 packet scope preflight：
  `pwsh -NoProfile -ExecutionPolicy Bypass -File scripts/check-task-packet.ps1 -PlannedTargetPaths <本轮准备改的路径>`
- `planned target paths / intended changed files` 是严格 task-packet 预检；普通 direct-dev 不因当前小包白名单阻断。
- `declared_changed_paths` 的硬匹配属于 strict preflight 阶段；默认 `scripts/check-task-packet.ps1` 只校验 task-packet 形状、source registration、roster alignment 和 forbidden paths。

## 5. 停机条件矩阵（冻结）

| 触发条件 | 停机级别 | 动作 |
|---|---|---|
| `validate-contracts.ps1` 失败 | P0 | 立即停机并转人工 |
| `run-governance-contracts.ps1` 失败 | P0 | 立即停机并转人工 |
| `run-golden.ps1` 失败 | P0 | 立即停机并转人工 |
| `check-final-gate.ps1` 失败 | P0 | 立即停机并转人工 |
| `lint-drift.ps1` 失败 | P1 | 停机，要求人工确认是否继续 |
| `doctor.ps1` 失败（环境/入口异常） | P1 | 停机，先修环境 |
| 任何外发相关资产被改动 | P0 | 立即停机 |
| 触及支付/交付/触达执行逻辑 | P0 | 立即停机 |
| 高限制字段放行策略被改动 | P0 | 立即停机 |
| 缺 task packet / 缺 declared changed paths | P0 | 立即停机 |
| declared `change_class` 低于 review gate matrix 计算结果 | P0 | 立即停机 |
| `MANDATORY_HUMAN_REVIEW` 缺 human review 或缺 required owner review | P0 | 立即停机 |
| 命中 `STOP_AND_ESCALATE` | P0 | 立即停机并转人工 |
| 实际改动路径超出 declared / allowed 范围 | P0 | 立即停机并转人工 |

连续失败规则：
- 同一脚本连续失败 >= 2 次，必须停机转人工。

## 6. 禁止自动推进的对象/能力/阶段

| 类别 | 禁止范围 | 说明 |
|---|---|---|
| 阶段 8 | 真实触达执行 | 仅允许 draft/preview，不得自动发送 |
| 阶段 9 | 真实支付/交付/退款执行 | 仅允许 draft/preview，不得自动执行 |
| 能力放行 | `EXTERNAL_RELEASE` | 外部软件 release 为 `CONTROLLED_OPENING`；未满足 task packet / release checklist / 审批 / 审计 / operator action 前不得 live |
| 高限制字段 | 自然人联系方式/治理补证 | 只能经人工审批链放行 |
| 对外承诺 | 任何对外能力承诺 | 禁止自动生成/修改 |

## 7. 自动化任务包范围规则（冻结）

| 规则 | 要求 |
|---|---|
| 单批次主题 | 仅允许单一阶段或单一主题（不得跨阶段杂糅） |
| 路径范围 | 必须显式声明 `declared_changed_paths` / `allowed_modification_paths` / `forbidden_modification_paths` |
| contracts 变更 | 必须以 D 文档为先导，且仅草案/补丁 |
| 触及高风险域 | 必须拆分成单独任务包并转人工 |

补充说明：
- 不再设置全仓统一 `<=10` 文件这类硬上限；
- 是否拆包主要由单主题要求、路径范围、review gate 与 stop conditions 共同决定；
- 触及控制资产时必须按 Batch Stop Rules 停机汇报。
- 提交前必须排除：`报告*.md`、`__pycache__/`、`*.pyc`、`.pytest_cache/`；
- 任务实现包完成后、提交前，应运行：`pwsh -NoProfile -ExecutionPolicy Bypass -File scripts/clean-python-cache.ps1`
- 临时报告文件与 Python 缓存产物不得混入 commit、review scope 或 task packet 实际改动清单。
