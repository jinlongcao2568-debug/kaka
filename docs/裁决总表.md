## 0. 文档任务

本文档解决以下问题：

1.  当前有哪些正式裁决已经被冻结；
2. 每个裁决属于哪份文档、位于哪一章、影响哪些实现面；
3. 哪些裁决属于正式裁决，哪些属于解释性裁决，哪些属于断言；
4. D1-D14 当前哪些文档已经完成逐条补号；
5. 页面、接口、导出、contracts、handoff、testing、governance 在实现时应回指哪条裁决；
6. 哪些裁决属于一票否决或高优先级阻断。

## 1. 裁决类型与编号规则

### 1.1 裁决类型

| 类型 | 编号前缀 | 含义 | 能否直接阻断实现 |
|---|---|---|---|
| 正式裁决 | `L0-R-*`、`D1-R-*`、`D2-R-*` … | 冻结正式边界、对象、阶段、门禁、消费优先级、交付或治理规则 | 可以 |
| 解释性裁决 | `L0-X-*`、`D13-X-*` 等 | 用于澄清语义、适用边界或防止误读，不单独推翻正式裁决 | 原则上不单独阻断 |
| 断言 | `L0-A-*` | 必须落实为 testing / gate / release / governance 的硬约束 | 可以 |

### 1.2 编号规则

- 裁决号一经分配不得重排；
- 允许追加，不允许改号覆盖旧义；
- 正式裁决与解释性裁决必须分前缀管理；
- 断言只允许登记为 L0 断言或明确声明的 D 文档断言；
- 本表登记的是**稳定引用面**，不是替代原文的简写版。

### 1.3 阻断级别

| 级别 | 含义 | 典型动作 |
|---|---|---|
| `BLOCKER` | 一票否决，冲突即不得合入或发布 | 阻断实现、阻断外发、阻断触达、阻断上线 |
| `HIGH` | 高风险冲突，必须在当前批次修复 | 限期修复、触发回归 |
| `MEDIUM` | 中等级偏差，允许继续编写但不得长期悬置 | 进入下一轮修复 |
| `LOW` | 风格、说明或引用层面的统一性问题 | 可集中修整 |

## 5. 当前索引维护规则

### 5.1 何时必须更新本表

以下任一情形发生时，必须同步更新本表：

1. 新增正式裁决或解释性裁决；
2. 新增 L0 或 D 文档断言；
3. 正式对象、正式枚举、公开边界、双闸门、字段治理、交付矩阵、发布门禁、模型放行规则发生变化；
4. 某文档从 `DRAFT` 升格为 `EFFECTIVE`；
5. 某文档发生 `SUPERSEDED` 替代关系。

### 5.2 不允许的维护方式

- 不得只在代码、注释、PR 描述或会议纪要中补裁决号；
- 不得修改旧号去承载新语义；
- 不得在本表中先登记“已生效”，但正文仍未同步；
- 不得让样例 JSON、API 示例、导出模板先引用未登记裁决号。

### 5.3 与正文的优先关系

- 本表只负责索引，不替代正文；
- 如本表与原文冲突，以原文为准；
- 但若正文已补号而本表未更新，则视为索引缺陷，必须修复。
