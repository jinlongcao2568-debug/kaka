## 0. 双线业务定位与输出边界

本文件是 `PRE_BID_PREDICTION` 投前预测线的辅助产品线文档，也是规则候选池，不是全系统核心商业主线。当前招投标业务方向以 `docs/业务方向_候选公示后证据包与投前预测双线契约.md` 和 `contracts/evaluation/business_direction_strategy_contract.json` 为准：候选公示后证据包分析是核心商业主线，投前预测分析是辅助产品线。

本文中的“主线能力层”只表示投前预测线内部优先落地的能力层，不等于全系统核心商业主线。投前线可以输出控标/定制标预测、废标风险、是否值得投、澄清或质疑建议；不能输出候选人核查结论、真实竞争者结论、围标/串标/陪标组合结论，也不能把刚发公告的招标文件 smoke 当成候选公示后证据包完成。

投前预测必须先通过时钟和流程门槛：只适用于近期 `02 招标文件公示`、`03 招标公告/关联公告`、`04 答疑澄清/补遗` 且投标截止日期、开标日期未过的项目。若同项目已经出现 `05 开标信息`，投前预测即不适用，应输出 `PRE_BID_NOT_ELIGIBLE_OPENING_STARTED` 并转入开标后核验或候选后证据包路线；若截止/开标已过，应输出 `PRE_BID_NOT_ELIGIBLE_DEADLINE_PASSED`。标准投前预测销售窗口为截止/开标前 168 小时以上；72-168 小时只能进入 `PRE_BID_LIMITED_FAST_REVIEW`；少于 72 小时应输出 `PRE_BID_NOT_ELIGIBLE_TOO_LATE_FOR_SALE`，不得包装成完整投前预测。

如果只有 `02/03` 而尚无 `04 澄清答疑/补遗`，预测只能标记为 `PREDICTION_BEFORE_CLARIFICATION`，属于半成品预测；后续出现澄清、答疑、补遗、变更或补充招标文件时，必须标记 `PREDICTION_RECALC_REQUIRED` 并重新下载相关文件做二次预测。出现 `04` 后才进入 `PREDICTION_AFTER_CLARIFICATION`。

投前线的下载和解析也必须按 `AnalysisStrategyPlan v1` 分层执行：优先解析招标公告、招标文件、答疑澄清中的资格条件、评分办法、技术参数、废标条款、付款履约和关键时钟字段；不默认解析投标文件公开、候选人资料、合同信息等后验材料。

## 1. 总口径

本草案不是资料摘录，而是投前预测、评审端风险检查、规则救济与履约结算风险的能力分层。

能力按四个层级组织：

| 层级 | 能力 | 定位 |
|---|---|---|
| 公告前机会与政策规则层 | 公告前机会、公平竞争审查、需求管理控标、政府采购本国产品政策、远程异地评标规则、AI机器审查、地区规则画像 | 把机会发现提前到采购意向/招标计划，并用正式政策规则增强控标和合规判断 |
| 主线能力层 | 项目筛选、控标预测、自评分、废标红线 | 仅指投前预测线内部优先能力；不替代候选公示后证据包核心商业主线 |
| 增强能力层 | 暗标、正偏离、授权签章、声明函、税率审计、电子监管 | 属于新增能力候选，适合后续扩展为投标文件审查 / 评审端检查能力 |
| 规则救济与履约结算层 | 异议质疑投诉、资格条件合法性、中标后合同与程序风险、工程结算审计风险、履约付款救济、专项整治与举报监督 | 独立扩展层，用于规则保护、合同履约和结算审计风险识别，不输出违法定性 |

除上述能力层外，还必须保留一组基础支撑能力。它们不单独作为主线能力、增强能力、公告前能力或规则救济能力命名，但会影响所有判断：

| 支撑项 | 作用 |
|---|---|
| 文件完整性与版本链 | 保证招标文件、附件、答疑、补遗、澄清、图纸、清单不漏看 |
| 资格/信用/业绩材料 | 保证“能不能投”和“能拿多少分”的前置判断成立 |
| 报价履约/付款风险 | 保证“值得投”和“中了能不能履约赚钱”的判断成立 |
| 复盘样本库 | 用废标公告、投诉公告、中标公告、流标公告反哺规则 |
| 项目来源库、AI采集与项目档案 | 用公告前线索、项目档案和内部情报整理支撑投前判断 |

输出口径必须保守：

- 可以输出“疑似风险”“命中线索”“建议核查”“建议质疑/澄清/放弃/谨慎投”。
- 不直接输出“内定”“违法已成立”“必然废标”“必然中标”。
- 官方案例、投诉决定、处罚案例只作为规则来源和相似案例，不把事后结论反向套到当前项目。
- 电子监管相关内容只能做合规提醒，不得提供规避监管、规避串标识别或绕过平台风控的方案。

评审端检查不能把所有问题都粗暴归为“废标”。同一细节在不同招标文件和评审场景下，可能对应不同结果层级：

| 结果层级 | 含义 | 产品表达 |
|---|---|---|
| 废标/无效响应 | 触发一票否决或实质性偏差 | 红线 |
| 扣分/不得分 | 不废标，但评分项无法得分 | 扣分风险 |
| 澄清/补正 | 需要按程序解释或补充说明 | 澄清风险 |
| 质疑/投诉失败 | 事后争议很难推翻原评审 | 事前复核优先 |

### 1.1 两法体系分流器

目标：先区分项目适用《政府采购法》体系还是《招标投标法》体系。两套体系在主体、资金来源、监管机构、采购方式、资格审查、提前发布制度和救济路径上不同，若前置分流错误，后续机会判断、控标判断、废标红线和救济时钟都会错。

分流不能做成简单二选一。政府采购工程、依法必须招标工程、非依法必须招标但参照招标规则的项目、国企/平台采购等混合场景，应先按正式项目分类字段归一，再落到具体规则路径。

| 分流项 | 判断 | 产品动作 |
|---|---|---|
| 正式采购制度字段 | 优先承接 `procurement_regime`、`procurement_category` 等正式分类字段 | 正式字段映射 |
| 法律体系 | 政府采购、政府采购工程、依法必须招标工程、非依法必须招标项目、国企/平台采购等适用规则不同 | 体系分类 |
| 资金来源 | 财政性资金、国有资金、社会资金等会影响监管和程序 | 资金来源识别 |
| 监管机构 | 财政部门、发展改革部门、住建/交通/水利等行业监管路径不同 | 监管路径识别 |
| 采购/招标方式 | 公开招标、邀请招标、竞争性谈判、竞争性磋商、询价、单一来源等规则不同 | 方式分流 |
| 资格审查主体 | 采购人、代理机构、评标委员会、资格审查委员会等职责不同 | 审查主体识别 |
| 提前发布类型 | 采购意向、招标计划、资格预审公告、招标文件提前公示等前置信号不同 | 前置信号分流 |
| 救济路径 | 政府采购通常走质疑/投诉，依法必招项目通常走异议/投诉 | 救济路径分流 |

可落地输出：

- `procurement_regime`。
- `procurement_category`。
- `legal_system_type`。
- `fund_source_type`。
- `regulator_type`。
- `pre_notice_type`。
- `qualification_reviewer`。

其中 `legal_system_type` 只是草案辅助标签，不替代 D2/D13 中的正式对象字段。

## 2. 公告前机会与政策规则层

公告前机会与政策规则层用于把机会发现提前到采购意向、招标计划、招标文件提前公示和政策规则变化。它不是替代公告后分析，而是为后续项目筛选、控标预测、自评分和救济判断提供前置信号。

### 2.1 公告前机会层

目标：在正式招标公告发布前，利用采购意向、招标计划和招标文件提前公示信息形成关注池。

| 规则 | 判断 | 产品动作 |
|---|---|---|
| 政府采购意向 | 项目名称、需求概况、预算金额、预计采购时间可形成早期机会信号 | 公告前关注 |
| 招标计划 | 工程招投标侧的计划发布可提前识别项目范围、时间窗口和潜在预算 | 计划入库 |
| 招标文件提前公示 | 可提前发现资格条件、评分办法、技术参数和合同条款风险 | 提前审查 |
| 采购意向到公告间隔 | 预计采购时间、公告时间和投标截止时间可形成准备窗口 | 时钟链扩展 |

可落地输出：

- `pre_notice_signal`。
- `procurement_intention_budget`。
- `expected_procurement_time`。
- `tender_plan_watchlist`。

### 2.2 公平竞争审查层

目标：把地方保护、所有制歧视、区域门槛和交易工具限制从普通资格合法性中拆出来，作为独立证据层。

| 风险类型 | 判断 | 产品动作 |
|---|---|---|
| 本地分支机构、本地纳税、本地社保 | 可能形成本地保护或区域限制 | 地方保护信号 |
| 本地业绩、本地奖项、本地协会证书 | 可能排斥外地供应商或放大本地供应商优势 | 区域壁垒检查 |
| 特定行业组织证书 | 若与项目履约必要性弱，可能形成非必要市场壁垒 | 证书必要性检查 |
| 按所有制、地区、产地差异评分 | 可能违反平等参与和公平竞争要求 | 差别待遇检查 |
| 指定交易工具、指定平台、指定机构 | 若缺少必要性，可能限制经营主体自主选择 | 交易工具限制检查 |

可落地输出：

- `fair_competition_barrier_type`。
- `local_protection_signal`。
- `ownership_discrimination_signal`。
- `market_barrier_remedy_path`。

### 2.3 需求管理控标层

目标：从采购需求形成过程识别控标源头，而不是只在招标文件参数发布后才判断。

| 规则 | 判断 | 产品动作 |
|---|---|---|
| 需求客观、明确、量化 | 采购需求不清或难以量化时，后续评分和履约争议风险增加 | 需求质量检查 |
| 达到规定金额或复杂程度的项目应开展需求调查 | 1000 万元以上货物/服务、3000 万元以上工程，以及技术复杂、社会关注度高等项目，应重点检查需求调查 | 需求调查触发 |
| 面向市场主体调查对象一般不少于 3 个 | 调查对象过少或单一，可能导致需求偏向特定供应商 | 调查充分性检查 |
| 同类业绩作为资格条件一般不超过 2 个 | 超量业绩门槛可能形成限制竞争 | 业绩数量限制风险 |
| 创新产品采购不得简单要求同类业绩、生产台数、使用时长 | 对新产品设置传统业绩门槛会抑制创新竞争 | 创新产品门槛检查 |

可落地输出：

- `demand_investigation_required`。
- `demand_objective_quantified`。
- `same_contract_count_limit_risk`。
- `innovation_no_past_performance_rule`。

### 2.4 政府采购本国产品政策层

目标：只在政府采购产品场景下，识别本国产品政策适用、价格评审优惠和声明责任风险。

| 规则 | 判断 | 产品动作 |
|---|---|---|
| 2026 年 1 月 1 日起实施政府采购本国产品政策 | 政府采购产品项目需识别本国产品、非本国产品和混合采购包 | 政策适用识别 |
| 本国产品相对非本国产品享受 20% 价格评审优惠 | 价格分测算应按政策适用条件模拟扣除后价格 | 价格扣除测算 |
| 多产品采购包需核查本国产品成本占比 | 达到政策要求时才可适用相应价格扣除 | 成本占比检查 |
| 声明函虚假 | 可能引发无效响应、追责或诚信风险 | 虚假声明风险 |
| 不得以所有制、品牌注册地、投资者国别等歧视供应商 | 本国产品政策不能被误用为供应商歧视工具 | 平等参与检查 |

可落地输出：

- `domestic_product_declaration_required`。
- `domestic_product_cost_ratio`。
- `domestic_price_deduction_20pct`。
- `false_declaration_liability`。

### 2.5 远程异地评标规则层

目标：把远程异地评标从一般“人为影响降低”的经验，升级为评审组织规则和系统风险检查。

| 规则 | 判断 | 产品动作 |
|---|---|---|
| 省级部门公布适用项目范围 | 远程异地评标是否适用不能凭经验猜测 | 适用范围识别 |
| 主副场责任 | 主场、副场职责和资料保存责任影响事后复核 | 主副场责任检查 |
| 专家库和评标工位共享 | 专家抽取和场所共享会影响评审组织方式 | 评审组织状态 |
| 系统和数据安全 | 远程评标增加系统故障、数据传输和资料完整性风险 | 系统风险提示 |
| 专家考核和回避 | 影响熟面孔评标、专家行为和主观分波动 | 专家管理信号 |
| 评审人员异常接触 | 业主代表、评标专家与投标人非正常沟通、利益往来或绕开回避要求，只能作为异常线索 | 接触线索提示 |
| 利益输送迹象 | 非法财务往来、异常咨询费、异常中介费等只作为合规风险线索，不做刑事定性 | 利益输送线索 |

可落地输出：

- `remote_cross_region_eval_applicable`。
- `main_sub_venue_responsibility`。
- `expert_pool_shared_state`。
- `remote_eval_system_failure_risk`。
- `owner_representative_eval_risk`。
- `expert_supplier_contact_signal`。
- `illegal_benefit_transfer_signal`。

### 2.6 AI评标与机器审查适配

目标：面向招标文件检测、投标合规自查、智能辅助评标、围串标识别、评标报告核验和投诉处理辅助等场景，提前把投标文件做成机器也能稳定解析、比对和回链的结构。AI 结论只能作为检测、提示和辅助，不替代招标人、代理机构、投标人、评标专家或监管部门的法定判断。

官方 AI 应用政策、公开平台检测规则和招标文件检测场景可作为政策/规则线索；投标文件自查、结构化响应、机器相似性检测属于投标文件内审或内部输入，不直接进入公开证据主链。

| 检查项 | 判断 | 产品动作 |
|---|---|---|
| 标书可读性 | 目录、标题、表格、页码、附件索引混乱会增加 AI 和人工评审成本 | 可读性评分 |
| 结构化响应 | 响应点、偏离点、证明材料、报价说明要能被机器抽取和定位 | 结构化响应评分 |
| 机器可检测相似性 | 技术方案语义相似、商务标关键报价特征相似、文件元数据相似会形成监管关注 | 相似性风险 |
| 投标文件合规自查 | 与招标文件逐项比对响应性、缺漏、错误、异常低价和证明材料 | AI 自查结果 |
| 评标报告核验 | 客观评审因素评分不一致、分值计算错误、专家打分偏离度过大应可被发现 | 评审核验提示 |
| 投诉处理辅助 | 投诉材料可被机器初筛，但不能替代行政监督部门处理结论 | 辅助审查边界 |

可落地输出：

- `ai_review_readability`。
- `machine_detectable_similarity`。
- `ai_bid_file_check_result`。
- `structured_response_score`。

### 2.7 地区规则差异与地方画像层

目标：把不同地区的投标规则侧重点沉淀为地方画像，用于投前工作量预估、评分拆解、报价策略和合规留痕。非正式经验材料只能作为经验观察，具体地区结论必须回到当地招标文件、公共资源平台规则、交易系统说明、专项整治公告或历史样本核验。

| 地区画像类型 | 经验观察 | 产品动作 |
|---|---|---|
| 暗标型地区 | 暗标格式、页数、字体、页眉页脚、身份痕迹、扣分/废标规则差异较大 | 暗标规则专项检查 |
| 低价型地区 | 价格权重或最低价倾向较强，报价策略和成本测算压力高 | 报价敏感性检查 |
| 细则型地区 | 评分细项密集、拆分颗粒度高，容易漏响应 | 评分矩阵逐项拆解 |
| 机器评标型地区 | 格式、字段、报价、资质、响应性更依赖系统化校验 | 机器可读性和符合性检查 |
| 高标准/高工作量地区 | 技术标页数、方案深度、附件数量和制作周期压力高 | 工作量和制作周期预估 |
| 专项整治型地区 | 围标串标、买标卖标、黄牛掮客等被重点治理，倒查风险上升 | 合规强审和留痕增强 |

地区经验表述要避免“某地一定好投/难投”。更稳妥的表达是“该地区历史样本中存在某类规则侧重点，建议按项目文件和当地平台规则复核”。

可落地输出：

- `regional_rule_profile`。
- `regional_bid_rule_type`。
- `dark_bid_rule_strictness`。
- `scoring_detail_density`。
- `low_price_region_signal`。
- `machine_eval_region_signal`。
- `regional_workload_multiplier`。
- `regional_enforcement_campaign_signal`。

## 3. 主线能力层

主线能力层包含四个能力：项目筛选、控标预测、自评分、废标红线。

### 3.1 项目筛选

目标：判断项目值不值得投入标书成本。

| 规则 | 判断 | 产品动作 |
|---|---|---|
| 场内标优先 | 公共资源交易中心、政府采购平台留痕更完整 | 加权 |
| 电子标优先 | 过程标准化，便于结构化和留痕 | 加权 |
| 暗标/盲评优先 | 供应商身份影响相对降低 | 加权 |
| 远程异地评标优先 | 评委本地关系影响相对降低 | 加权 |
| 大城市、营商环境好地区优先 | 规则透明度和公开材料质量通常更高 | 加权 |
| 本地小金额项目适合新手 | 沟通和履约成本低，适合积累业绩 | 加权 |
| 新手或低成熟投标团队 | 第一目标应是跑通流程、降低废标、积累业绩和模板，不宜重仓大项目 | 降低投入上限 |
| 流标、二次招标、重新招标、投标人数不足三家 | 可能存在补位机会 | 重点关注 |
| 询价、竞争性谈判、竞争性磋商 | 流程快，适合低成本尝试，但需看门槛 | 重点关注 |
| 盲投采购方式分层 | 询价、竞争性谈判更适合低成本盲投；公开招标、竞争性磋商要拆评审表后再判断 | 采购方式分层 |
| 主观分占比过高 | 主观分过高会放大评审不确定性，盲投胜率下降 | 主观分降权 |
| 客观分可得性 | 证书、业绩、人员、产品参数等客观分可拿多少，决定是否值得盲投 | 客观分测算 |
| 价格分空间 | 价格分权重、最高限价、市场价格和成本空间共同决定盲投可行性 | 价格空间测算 |
| 盲投团队产能 | 项目池、标书制作产能、需求/商务/方案/产品/价格分工影响可投数量 | 团队产能检查 |
| 投前反馈策略 | 在合法澄清、异议或采购文件反馈窗口内，先评估主体资格、证据强度和商业策略披露边界 | 反馈/救济策略评估 |
| 场外开标 | 留痕弱、人为空间大 | 降权或 PASS |
| 纸质标 | 制作和提交风险高，结构化难度高 | 降权 |
| 付款极差 | 中标后现金流风险大 | 降权 |
| 明显被提前锁定 | 与控标预测联动 | 谨慎投或 PASS |

可落地输出：

- `project_roi_precheck`：投前 ROI 初筛。
- `project_selection_reasons`：建议投、谨慎投或放弃的原因。
- `watchlist_reason`：流标重招、二次开标、小金额、本地项目等机会标签。
- `blind_bid_pipeline_stage`。
- `bid_team_capacity`。
- `document_feedback_strategy`。
- `exposure_risk_before_bid`。

### 3.2 控标预测

目标：识别招标文件是否存在疑似量身定制、指向性、排他性或资料不对称风险。

#### 2.2.1 通用控标信号

| 规则 | 判断 |
|---|---|
| 资质、业绩、人员、检测报告、产品参数、服务网点刚好只匹配某一家 | 高风险 |
| 单条参数合理，多条组合后只剩一两家可满足 | 高风险 |
| 参数与某厂家官网、彩页、产品手册高度相似 | 高风险 |
| 要兼容历史系统但不提供接口文档或开放标准 | 中高风险 |
| 现场演示、样品、讲标分值过高且标准主观 | 中高风险 |
| 交付期明显短于正常采购/生产周期 | 提前备货风险 |
| 评分项集中匹配某供应商优势 | 中高风险 |
| 非核心指标高分化、主观化 | 中高风险 |
| 厂家授权被设置为资格、实质性要求或高分项且缺少替代路径 | 中高风险 |
| ISO、CMA、检测资质、行业证书与采购需求关联弱但权重高 | 中高风险 |
| 现场踏勘被设置为资格条件、实质性条件或加分条件 | 中高风险 |

#### 2.2.2 政府采购类信号

| 信号 | 风险解释 | 证据需求 |
|---|---|---|
| 参数、检测报告、证书、品牌倾向过强 | 可能指向特定产品或供应商 | 条款原文、市场产品对比、厂家资料 |
| 售后网点、本地化要求异常 | 可能排斥外地供应商 | 条款必要性、项目服务半径说明 |
| 评分细则过细 | 可能把某家优势拆成得分点 | 评分项、权重、竞品可满足情况 |
| 固定机构检测报告 | 可能形成非必要门槛 | 法定强制性依据、替代机构可行性 |
| 厂家授权、ISO、CMA 等证明材料机械设为门槛 | 可能与采购需求无直接关系，或形成排他授权 | 项目必要性、强制依据、替代证明路径 |
| 现场踏勘与投标资格或得分挂钩 | 可能排斥无法到场或外地供应商 | 踏勘必要性、是否存在合规依据 |

#### 2.2.3 工程类信号

| 信号 | 风险解释 | 证据需求 |
|---|---|---|
| 资质等级过高 | 可能超过项目实际需要 | 项目规模、法规资质要求、同类项目对比 |
| 项目经理/技术负责人条件过细 | 可能卡特定团队 | 证书、业绩、社保、在建条件 |
| 类似业绩限定过严 | 特定区域、特定类型、特定金额、特定时间组合 | 历史同类招标文件、市场供应商数量 |
| 奖项、信用评价、本地协会荣誉 | 可能形成区域保护或非必要加分 | 奖项权威性、必要性、可替代性 |
| EPC/设计施工总承包 | 设计、初设、方案、技术路线可能提前绑定 | 初设来源、方案前置、评定分离机制 |

#### 2.2.4 服务类信号

| 信号 | 风险解释 | 证据需求 |
|---|---|---|
| 团队配置过细 | 可能绑定特定服务商人员结构 | 人员必要性说明、同类服务对比 |
| 驻场、响应时间、本地服务能力过严 | 可能排斥非本地服务商 | 服务内容、响应半径、替代服务方式 |
| 案例经验要求过细 | 可能只匹配某供应商历史项目 | 业绩维度拆解、市场可满足情况 |
| 过往项目证明材料多重叠加 | 合同、发票、验收、用户证明全要可能形成高门槛 | 证明必要性和评分权重 |

#### 2.2.5 控标风险的缓释和变数

疑似控标不等于必然放弃。系统应同时识别“风险信号”和“变数信号”，避免把所有高风险项目都机械判为 PASS。

| 变数信号 | 判断 | 产品动作 |
|---|---|---|
| 暗标、盲评、远程异地评标 | 现场引导和身份影响相对降低 | 风险适度缓释 |
| 监管高压、投诉风险明显 | 过度定向条款更容易被关注 | 建议澄清或质疑评估 |
| 对手废标概率高 | 即使存在倾向，也可能因材料错误产生机会 | 联动废标红线 |
| 甲方内部意见可能不一致 | 定标和履约侧存在不确定性 | 不输出确定性结论 |
| 流标重招或二次招标 | 原有锁定结构可能松动 | 进入关注池 |

可落地输出：

- `tailored_bid_risk_profile`。
- `restrictive_clause_hit_list`。
- `market_substitutability_check`。
- `suspicious_match_to_vendor_material`。

### 3.3 自评分

目标：按评分办法测算我方材料能拿多少分，判断是否值得继续投入。

| 规则 | 判断 | 产品动作 |
|---|---|---|
| 评分办法必须拆成资格、技术、商务、价格、政策加分 | 不能只看是否能报名 | 评分结构化 |
| 评分项对应我方材料库 | 有证据才算可拿分 | 自评分 |
| 理论能投但关键加分项拿不到 | 实际陪跑概率高 | 谨慎投 |
| 技术商务分高、价格分低 | 证据优势比低价更重要 | 强调材料准备 |
| 价格分高 | 需要精算价格策略 | 联动报价风险 |
| 小微企业预留、价格扣除、国产产品政策 | 中小企业机会 | 政策加分提示 |
| 同品牌/核心产品规则 | 单一产品同品牌、非单一产品核心产品同品牌会影响有效竞争家数和候选资格 | 品牌有效投标人测算 |
| 资格条件被作为评分因素 | 资格条件不得重复作为评审因素放大优势 | 评分合法性检查 |
| 评分因素未细化量化 | 评分项应对应商务条件和采购需求，不能用笼统主观项扩大裁量 | 评分量化检查 |
| 客观但不可量化指标作为评分项 | 更适合作为实质性要求，不宜作为评分项 | 非量化评分风险 |
| 评定分离项目 | 不能只盯评标分，还要看定标因素 | 定标规则抽取 |
| 候选人不排序、票决法、集体议事法 | 进入定标环节后，价格、技术方案、信誉、团队实力仍可能改变结果 | 定标因素结构化 |
| 主副场、远程异地、双盲、专家抽取和回避规则 | 评审组织方式会影响现场沟通和身份影响空间 | 评审组织规则抽取 |
| 定标过程留痕、密封存档要求 | 事后争议通常看程序和记录，不是重新比较投标文件 | 定标程序风险提示 |
| 评分项没有对应证明材料 | 理论响应不等于实际得分 | 材料缺口提示 |

自评分还应结合评审端视角，重点降低评委阅读成本：

| 经验 | 判断 | 产品动作 |
|---|---|---|
| 专家先看目录和左侧导航 | 目录层级深、逻辑闭环会提高专业感 | 目录结构检查 |
| 专家只看打分点证书 | 无关证书、荣誉、简介价值低 | 证书与评分项映射 |
| 0.1 分、0.01 分都可能决定中标 | 小承诺、小证明、小正偏离不能忽略 | 得分细项提示 |
| 方案有关键词、图表、第一手资料、现场痛点、本地服务资源 | 能证明不是模板化响应 | 方案质量评分 |
| 评分项就是目录 | 小标题对应评分点，证明材料前置 | 得分点导航检查 |
| 每个得分点要有标题、响应、方案、证明、页码、附件索引 | 评委不应被迫在几百页里找证据 | 证据可达性检查 |
| 评分索引和响应页码 | 每个评分项应能快速定位到响应内容和证明材料页码 | 评分索引检查 |
| 证据链目录 | 合同、发票、验收、证书、检测报告等应按得分点组织，不只按材料类型堆放 | 证据链目录检查 |
| 技术偏离定位 | 技术偏离、参数响应和正偏离证明要能被快速定位 | 技术定位检查 |
| 商务证明定位 | 业绩、人员、社保、信用、售后、本地服务证明要能被快速定位 | 商务定位检查 |
| 报价说明定位 | 总价、分项、折扣、税率、异常低价说明和成本证据要能被快速定位 | 报价定位检查 |
| 公司简介、无关荣誉和泛化模板堆砌 | 增加页数但不增加得分确定性 | 无效内容降权 |
| 采购人代表可能影响主观分 | 同一材料在不同评审组织下存在主观波动 | 评审端不确定性标记 |
| 专家可能先形成潜在中标候选，再回看材料确认 | 目录、首屏、关键词和核心证明会影响初始判断 | 首轮可读性检查 |
| 远程异地双盲下材料要能离开人独立得分 | 不能指望现场解释、关系沟通或代理引导 | 证据自解释检查 |
| 打分结束后通常以复核、质疑、投诉为主，不应指望重新认真打分 | 投前材料质量比事后解释更重要 | 事前复核优先 |

可落地输出：

- `bid_score_forecast`。
- `winnability_band`：高/中/低。
- `score_gap_reason`：业绩缺口、人员缺口、证书缺口、价格压力、政策不适用。
- `core_product_brand_map`。
- `same_brand_effective_bidder_count`。
- `same_brand_candidate_rule`。
- `qualification_as_score_factor_risk`。
- `score_factor_quantification_check`。
- `non_quantifiable_score_item_risk`。
- `score_index_completeness`。
- `evidence_chain_navigation_score`。
- `technical_deviation_locator_state`。
- `commercial_evidence_locator_state`。
- `price_explanation_locator_state`。

### 3.4 废标红线

目标：从招标文件、格式文件、答疑补遗和投标材料要求中抽取一票否决和高频废标风险。

高频关键词：

`承诺`、`无效`、`废标`、`不得`、`必须`、`签字`、`盖章`、`偏离`、`声明函`、`信用`、`社保`、`保证金`、`有效期`、`暗标`、`页数`。

| 规则 | 判断 | 产品动作 |
|---|---|---|
| 先看废标条款 | 废标是一票否决 | 红线清单 |
| 识别明确的实质性要求 | 政府采购中不是采购文件每句话都能当然废标，应看资格条件、星号条款、不可偏离条款、符合性审查表和无效响应条款 | 实质性要求抽取 |
| 法人章和法定代表人章不能混 | 必须按文件定义 | 签章检查 |
| 授权书、投标函、承诺书、报价表、偏离表逐项检查 | 缺签缺章高频废标 | 表单检查 |
| 中小企业声明函真实且匹配 | 企业类型、行业、制造商/承接主体不能错 | 声明函检查 |
| 业绩材料完整 | 合同、发票、验收、中标通知、用户证明按要求组合 | 业绩证明检查 |
| 社保材料匹配 | 时间、人员、单位、证明形式 | 人员证明检查 |
| 信用查询提前做 | 信用异常可能资格不通过 | 信用预警 |
| 报价大小写、合计、税率、分项报价一致 | 报价错误高风险 | 报价校验 |
| 暗标不得出现身份识别信息 | 公司名、LOGO、水印、联系人、人员明显身份、文件属性 | 暗标检查 |
| 页数限制 | 限页时代不能堆材料 | 页数检查 |
| 页码格式和分册页码规则 | 连续页码、分册起止、目录页码要按文件要求 | 页码检查 |
| 彩色扫描件、拍照件、复印件要求不一致 | 黑白复印件加盖公章不一定等同于彩色扫描件 | 扫描件形式检查 |
| 证书有效期不足 | 有些文件要求覆盖投标期，稳妥场景还要看履约期 | 证书有效期检查 |
| 同品牌导致有效投标人不足 | 政府采购货物服务中，同品牌或核心产品同品牌可能按一家计算 | 有效竞争家数检查 |
| 多标包连带否决 | 一个标包超限价或出问题，不当然导致全部标包无效，除非招标文件明确约定 | 标包独立性检查 |
| 竞争性谈判/磋商二次报价规则 | 两家能否继续、二次报价能否高于首次、能否转单一来源，都要按采购方式、法规依据和采购文件判断 | 采购方式规则检查 |
| 补遗公告漏看 | 补遗可能改资格、评分、格式、开标时间、报价规则 | 版本链检查 |
| 原件备查未准备 | 开标现场可能需要原件 | 原件清单 |

可落地输出：

- `fatal_rejection_checklist`。
- `signature_seal_matrix`。
- `format_requirement_checklist`。
- `material_completeness_checklist`。

## 4. 基础支撑层

基础支撑层不改变能力口径，但不能从文档中删除。它为主线能力层和增强能力层提供输入、校验和复盘依据。

### 4.1 文件完整性与版本链

目标：确保分析基于完整、最新、可回链的招标文件包。

| 规则 | 判断 | 产品动作 |
|---|---|---|
| 必须保存软件版文件 | PDF 图片或截图不利于搜索和结构化 | 标记解析质量 |
| 招标文件、附件、图纸、清单、答疑、补遗都要入链 | 补遗可能改变资格、评分、截止时间 | 生成文件完整性清单 |
| 答疑/澄清版本优先级必须识别 | 后发版本可能覆盖原文件 | 生成版本链 |
| 关键附件缺失 | 例如图纸、接口文档、CAD、清单缺失 | 进入资料不对称风险 |
| 时间字段必须区分 | 公告发布时间、报名截止、投标截止、开标时间、质疑/异议截止 | 生成时钟链 |

读文件顺序：

| 顺序 | 要看什么 | 原因 |
|---:|---|---|
| 1 | 废标条款 | 一票否决，先判断红线 |
| 2 | 资格条款 | 先确认能不能入场 |
| 3 | 评分办法 | 再判断实际能拿多少分 |
| 4 | 格式文件 | 确认签字、盖章、声明函、偏离表、投标函、报价表怎么填 |
| 5 | 答疑、补遗、澄清 | 关键要求可能不在原文件，后发版本可能覆盖前文 |

可落地输出：

- `document_completeness_state`。
- `notice_version_chain`。
- `clock_chain_profile`。
- `missing_attachment_review_request`。

### 4.2 资格/信用/业绩材料

目标：判断是否具备入场资格，以及评分项是否有足够证明材料。

| 规则 | 判断 | 产品动作 |
|---|---|---|
| 资质等级是否满足 | 工程类重点看企业资质等级、专业类别 | 资格缺口 |
| 项目经理/技术负责人是否满足 | 证书、专业、等级、在建、社保 | 人员缺口 |
| 类似业绩是否满足 | 类型、金额、时间、区域、证明材料 | 业绩缺口 |
| 业绩时间口径是否正确 | “近三年”等口径要按招标文件定义，不宜自行按自然年理解 | 业绩时间检查 |
| 业绩最终用户是否清楚 | 中间商、代理或平台合同若最终用户不清，可能不加分或被质疑 | 最终用户检查 |
| 母子公司/母公司、子公司资质和业绩 | 独立法人之间不能天然互用，除非依法组成联合体且招标文件允许 | 法人边界检查 |
| 总公司、分公司业绩和主体 | 分公司不是独立法人，但仍要看招标文件允许主体、合同主体、纳税社保主体和履约责任 | 总分公司主体检查 |
| 合法分包业绩 | 需有总包合同、分包合同、甲方认可、验收、付款和实际履约范围 | 分包业绩证明链 |
| 非法转包、违法分包业绩 | 不能稳当地作为有效业绩，后续可能引发质疑或诚信风险 | 业绩有效性风险 |
| 社保材料是否满足 | 时间、单位、人员、证明形式 | 资格/加分风险 |
| 派遣工、全员社保、五险完整性 | 文件要求全员或关键人员社保时，要能证明实际劳动关系和缴纳完整性 | 社保细项检查 |
| 信用限制是否命中 | 信用中国、政府采购严重违法失信、失信被执行人、重大税收违法 | 资格阻断或复核 |
| 信用主体是否查全 | 公司、法定代表人、项目经理、关键人员、关联企业可能都要核查 | 信用主体矩阵 |
| 项目经理无在建承诺 | 不能只承诺，必要时要查公开平台或证明材料 | 无在建核验 |
| 法人或企业信息变更后证照未同步 | 资质证书、许可证、声明主体可能与现状不一致 | 证照一致性检查 |
| 保证金、有效期、联合体限制 | 容易被忽略但可能阻断 | 红线提示 |
| 业绩材料完整 | 合同、发票、验收、中标通知书、用户证明按要求组合 | 业绩证明检查 |

业绩材料应提前标准化保存：

- 合同关键页；
- 金额页；
- 签章页；
- 发票；
- 验收报告；
- 中标通知书；
- 用户评价；
- 项目照片；
- 服务记录。

可落地输出：

- `qualification_gap_profile`。
- `supplier_material_gap_list`。
- `credit_precheck_result`。
- `performance_evidence_library`。

### 4.3 报价履约/付款风险

目标：避免只看中标概率，不看履约和现金流。

| 规则 | 判断 | 产品动作 |
|---|---|---|
| 低价中标可能带来履约风险 | 无利润报价容易被关注，也可能拖累公司 | 异常低价提示 |
| 政府采购货物服务异常低价 | 报价明显低于其他通过符合性审查报价且可能影响质量或诚信履约时，应先要求书面说明和证明；不能证明合理性才可能无效 | 异常低价说明触发 |
| 财库〔2026〕2号异常低价触发线 | 低于全部通过符合性审查供应商平均报价 50%、低于次低报价 50%、低于最高限价 45%，或评委专业判断过低时，应启动审查 | 触发线检查 |
| 财库〔2026〕2号触发线调整 | 采购人可结合项目提高前三项数值标准，但最高不超过 65%，以采购文件和审查记录为准 | 触发线调整检查 |
| 异常低价说明时间 | 政府采购异常低价审查中，供应商现场提交成本说明和证明材料的合理时间一般不少于 30 分钟 | 说明时间提醒 |
| 成本证据包 | 原材料、人工、制造费用、同类成交价、市场价格、行业成本等材料要提前准备 | 成本证据包 |
| 低价审查记录 | 启动原因、审查意见、审查结果、供应商说明和证明材料应归档 | 评审记录检查 |
| 工程招投标低价 | 重点看是否低于成本和履约可行性，不能与政府采购货物服务的异常低价说明程序混用 | 低于成本风险 |
| 价格分高要精算 | 价格决定胜率 | 价格敏感性分析 |
| 技术商务分高要看证据优势 | 低价未必有效 | 材料优势提示 |
| 付款周期差 | 中标后现金流风险 | 履约降权 |
| 成本、利润、税费、人工、材料、回款都要算 | 不能只看中标 | 履约可行性 |
| 工程类不平衡报价 | 评标阶段可能占优，但结算审计可能因工程量变化、清单偏差被调整或扣减 | 结算敏感性分析 |
| 固定单价合同 | 固定单价不等于结算无风险，仍要看合同条款、清单偏差、工程量变化和审计口径 | 合同结算风险 |
| 投标承诺和履约资料 | 中标只是开始，后续结算更依赖过程资料和可核验履约证据 | 过程资料提醒 |

可落地输出：

- `price_strategy_warning`。
- `payment_risk_profile`。
- `delivery_profitability_check`。
- `settlement_audit_risk_profile`。
- `abnormally_low_bid_explanation_required`。
- `low_price_evidence_package`。
- `quality_delivery_impact_check`。
- `abnormal_low_price_trigger`。
- `cost_breakdown_ready`。
- `materials_labor_cost_evidence`。
- `low_price_review_record`。

### 4.4 复盘样本库

目标：把投前预测和评审端检查沉淀成可校准的样本库。

| 复盘来源 | 学什么 | 产品化用途 |
|---|---|---|
| 中标公告 | 对手、报价、候选排序、得分结构 | 竞争画像、价格区间 |
| 废标公告 | 别人死在哪里 | 废标规则库 |
| 投诉公告 | 哪些条款容易被质疑 | 控标样本库 |
| 流标公告 | 投标人数、废标原因、是否二次招标 | 捡漏机会 |
| 评分结果 | 技术/商务/价格差距 | 胜率模型校准 |

可落地输出：

- `rejection_case_library`。
- `complaint_clause_library`。
- `competitor_score_history`。
- `rebid_watchlist_state`。

### 4.5 提交与开标流程

目标：把“文件做对”延伸到“能按时、按规则、可留痕地提交和开标”。这部分不是控标识别，但会直接影响废标和无效响应风险。

| 规则 | 判断 | 产品动作 |
|---|---|---|
| 电子标提前一到两天上传 | 系统、CA、浏览器、驱动、网络、文件大小、签章和加密都可能出问题 | 上传时限提醒 |
| 用制作/上传环境参加开标 | 减少临时换电脑导致的 CA、驱动、浏览器问题 | 开标环境检查 |
| 竞争性磋商、竞争性谈判可能有二次报价 | 上传完不等于结束，漏报可能无效响应或价格分为零 | 二次报价提醒 |
| 线下开标资料要带齐 | 公章、身份证、授权委托书、CA、原件备查材料、备用电脑 | 现场清单 |
| 上传、签章、加密、解密记录要留存 | 便于事后复核系统异常或提交争议 | 操作留痕 |

可落地输出：

- `submission_opening_risk_hits`。
- `opening_material_checklist`。
- `second_quote_reminder`。

### 4.6 项目来源库、AI采集与项目档案

目标：把“看到公告后再判断”前移到项目生命周期早期线索发现、结构化采集和持续跟进。AI 的定位是辅助收集、清洗、分类、整理和生成预览，不替代项目判断、证据核验或投标决策。

项目来源库应覆盖多渠道，不只依赖招标网站会员：

| 来源渠道 | 可发现的信息 | 产品动作 |
|---|---|---|
| 专业招标网站 | 招标公告、中标公告、项目聚合信息 | 多平台对比和去重 |
| 政府采购网、公共资源交易平台 | 采购意向、需求意见征集、公告、补遗、成交结果 | 正式公开链优先 |
| 重大项目清单 | 年度重点项目、新建、续建、前期项目 | 项目生命周期分层 |
| 发改、住建、自然资源等主管部门平台 | 审批、立项、用地、规划、施工许可等早期信号 | 公告前线索入池 |
| 土地招拍挂 | 工业、商业、住宅用地成交后的建设和采购机会 | 土地后续机会反推 |
| 地方新闻 | 签约、开工、奠基、封顶、竣工等项目节点 | 时间节点补强 |
| 设计院、咨询公司中标信息 | 可研、设计、施工图之后的施工、设备、家具、系统采购机会 | 上游中标反推 |
| 细分行业官网 | 教育、医疗、金融、国企、上市公司采购平台线索 | 行业专题来源登记 |
| 上下游反推 | 家具盯装修、装修盯土建、设备盯建设进度 | 产业链机会追踪 |
| 圈子/社群信息 | 协会、商会、供应商群等快速消息 | 弱线索登记，需公开回链 |

AI 采集不应先做万能爬虫，应先明确需求和输出：

| 步骤 | 要点 | 产品动作 |
|---|---|---|
| 拆需求 | 明确项目名称、地区、建设性质、法人单位、建设内容、投资规模、建设期限、行业分类等字段 | 采集字段模板 |
| 定格式 | 输出统一表格，便于导入系统分析 | 表格版输出 |
| 定分类 | 医疗、教育、产业、交通、文旅等项目分类 | 分类标签 |
| 生成提示词 | 把搜索规则、筛选规则、输出格式写清楚 | AI 采集提示词 |
| 输出预览 | 表格版用于分析，网页/预览版用于查看和分享 | 双版本输出 |
| 复盘固化 | 记录高质量平台、有效关键词、分类规则调整 | 来源质量反馈 |

重点项目应形成项目档案夹，而不是看到线索就直接判断是否投标：

| 档案模块 | 主要内容 | 用途 |

## 5. 增强能力层

增强能力层包含六个能力：暗标、正偏离、授权签章、声明函、税率审计、电子监管。

这些能力大多需要消费投标文件、企业内部材料或电子投标过程信息。按当前 L0/D 文档口径，它们不能直接混入现有公开证据主链，必须作为新增能力候选治理。

### 5.1 暗标

目标：检查暗标是否存在身份泄露、版式违规、元数据痕迹和内容识别风险。

暗标检查不能只查公司名，应覆盖版式、标识、痕迹、内容和元数据。

| 类型 | 高频风险点 |
|---|---|
| 版式 | 字体、字号、颜色、页边距、行间距、首行缩进、空行 |
| 标点符号 | 英文标点、异常符号、公式字体、特殊符号 |
| 页面元素 | 页眉页脚、页码、目录、图表对齐、表格标题 |
| 隐性痕迹 | 隐藏目录、空格替代空字符、模糊特征、文件属性 |
| 身份信息 | 人员名字、企业名称、项目简称、特殊措辞、水印 |
| 限页要求 | 技术标限页、附件限页、目录和封面是否计页、超页处理规则 |

高分暗标要素：

| 要素 | 检查点 |
|---|---|
| 有效厚度 | 方案完整、材料充分、细节多 |
| 第一手资料 | 现场调研、原始设计资料、项目现状 |
| 文章架构 | 目录深、章节完整、逻辑闭环 |
| 采分点覆盖 | 评分办法全覆盖，可有超额采分点 |
| 图表化率 | 图表专业且解释问题 |
| 项目痛点 | 能讲出甲方痛点 |
| 本地服务 | 人员、仓储、响应、备件、路线、驻场、应急 |
| 短准密 | 限页场景下每页都要服务评分，删掉泛化模板和无关宣传 |

可落地输出：

- `dark_bid_format_check`。
- `dark_bid_identity_trace_check`。
- `document_metadata_risk_check`。
- `dark_bid_quality_score`。

### 5.2 正偏离

目标：判断正偏离是否可量化、可验证、可比较、可履约，避免空话承诺。

商务正偏离候选：

| 方向 | 可写内容 |
|---|---|
| 有效期 | 投标有效期延长、授权期限覆盖 |
| 工期 | 提前规划交接、一次验收、备品备件提前移交 |
| 地点 | 扩大免费服务区域、免费二次搬运、多地点实施不加价 |
| 售后 | 质保期延长、响应时间缩短、驻场服务、培训增加 |
| 报价 | 让利承诺、锁定总价不调整、放弃额外索赔、免费差旅、报价无漏项 |
| 团队 | 项目经理、项目团队、业绩、资质、信用记录增强 |
| 履约保证金 | 保证金缴纳、退还、替代担保方式更有利但可履约 |
| 主体资格 | 法人资格、联合体、分包、纳税社保、财务状况更优 |
| 权利义务 | 知识产权、不侵权承诺、不可抗力配合机制更清楚 |
| 样品演示 | 样品、演示、培训、验收支持更完整 |

技术正偏离候选：

| 方向 | 要求 |
|---|---|
| 性能、扩展性、安全性、生态性、合规性、国际化、用户体验 | 必须可量化、可验证、可比较 |
| 参数方向 | 不能乱用大于号/小于号，指标方向必须符合采购需求 |
| 证明材料 | 虚正偏离不加分，必须有参数、报告、证书、方案或履约资源支撑 |

可落地输出：

- `positive_deviation_quality_check`。
- `positive_deviation_evidence_map`。
- `parameter_direction_check`。

### 5.3 授权签章

目标：检查授权委托、签字盖章、投标函和报价表是否存在评审端风险。

授权委托书：

| 风险点 | 判断 | 产品动作 |
|---|---|---|
| 人名章代替真实签字 | 文件不认可或无备案时可能授权无效 | 签署形式检查 |
| 只写全权代理 | 授权事项不具体，应明确参加投标、签署文件、澄清、报价、领取通知等事项 | 授权范围检查 |
| 漏写无转委托权 | 可能形成授权缺陷 | 授权条款检查 |
| 授权期限不足 | 应覆盖并最好超过投标有效期，保守做法是留出冗余期 | 授权期限检查 |
| 机打签名、抠图签名 | 不能证明本人真实意思表示，后续风险高 | 签名真实性风险提示 |
| 真实签署过程缺失 | 手写签字扫描、电子签署记录、授权审批记录应能回溯 | 签署证据留存 |
| 身份证复印件未按要求盖章 | 附件形式不合规 | 身份证附件检查 |

投标函和报价表：

| 风险点 | 判断 | 产品动作 |
|---|---|---|
| 投标函擅自改模板 | 可能形成实质性偏差 | 模板保真检查 |
| 报价一览表和明细表不对应 | 总价、分项、税率、数量、单位、优惠、折扣冲突 | 报价一致性校验 |
| 小数位不符合要求 | 例如要求保留两位小数 | 小数位检查 |
| 折扣只写百分比 | 必要时应落到具体金额 | 折扣金额化提示 |
| 工期、地点、税率、有效期前后不一致 | 可能废标或扣分 | 字段一致性检查 |
| 日历日和天混用 | 工期、有效期、服务期应按文件原话 | 时间单位检查 |
| 二次报价漏报 | 可能无效响应或价格分为零 | 流程提醒 |

其他签章材料：

| 风险点 | 产品动作 |
|---|---|
| 签字盖章要求同时满足 | 签章矩阵 |
| 盖章位置错误 | 公章通常应盖在公司名称、落款或文件要求位置 | 盖章位置检查 |
| 身份证正反面放错 | 身份证附件检查 |
| 保证金备注缺项目包号 | 保证金备注检查 |
| 保证金多付或错付 | 保证金金额检查 |
| 水印复用旧项目资料 | 水印风险检查 |
| 查询件替代原始证书 | 证明效力检查 |

高危水印包括：

- 上一个项目名称；
- 仅限备案使用；
- 再次复印无效；
- 项目编号或包号错误；
- 遮挡证书关键信息；
- 只限某分公司使用但由总公司投标；
- 项目简称、客户简称或用途说明暴露旧资料复用。

### 5.4 声明函

目标：检查政策声明类文件是否真实、适用、主体一致。

中小企业声明函：

| 检查项 | 风险 |
|---|---|
| 标题名称、行业分类、从业人员、营业收入、资产总额、数据单位、划型结论、盖章主体、声明主体 | 任一错误都可能导致声明不成立或虚假声明风险 |
| 行业分类 | 应对应采购标的所属行业，不能凭营业执照随意选 |
| 数据单位 | 万元、元、人写错会放大风险 |
| 联合体、分包、制造商场景 | 声明主体和盖章主体尤其要一致 |
| 辅材覆盖 | 文件要求覆盖辅材时，不能只写主材或核心产品 |

本国产品/进口产品声明：

| 检查项 | 风险 |
|---|---|
| 招标文件是否允许进口产品 | 未允许时投进口产品可能无效 |
| 国产和进口混投 | 可能导致投标无效 |
| 进口产品按国产填声明 | 可能构成虚假材料风险 |
| 本国产品成本占比、优惠比例 | 不能写死全国规则，必须按现行政策和招标文件核验 |
| 证明材料 | 营业执照、厂房租赁、ISO、软著/专利、发票、物料清单等 |

其他承诺和声明：

| 检查项 | 风险 |
|---|---|
| 无诉讼、无仲裁、无重大违法等承诺 | 不能直接写“无”，应查询并留证 |
| 无行贿犯罪记录 | 通常至少要查企业和法定代表人，文件另有要求时按要求扩展 |
| 信用、纳税、社保承诺 | 承诺内容应与查询截图、缴纳证明、审计或财务材料一致 |
| 声明函主体和项目包号 | 项目名称、项目编号、包号、采购标的要与招标文件一致 |

- `declaration_form_risk_hits`。
- `sme_declaration_check`。

### 5.5 税率审计

目标：检查税率、审计报告、财务数据和异常低价说明是否一致、可验证、可履约。

税率：

| 检查项 | 风险 |
|---|---|
| 税率和征收率混淆 | 影响报价实质性、合同履约、发票开具 |
| 财务未参与税率确认 | 做标人员凭经验容易填错 |
| 报价表税率、分项、总价不一致 | 可能影响报价实质性 |

审计报告：

| 检查项 | 风险 |
|---|---|
| 审计报告签字盖章、关键页缺失 | 可能废标或扣分 |
| 财务报表勾稽关系冲突 | 资产负债表、利润表、现金流量表、附注数据不能矛盾 |
| 标书财务数据与审计数据冲突 | 可信度下降 |
| 二维码、验证码、查询方式不可验证 | 审计报告可信度下降 |
| 负债、诉讼、担保、异常指标 | 需要解释和风险可控证明 |

异常低价：

本小节异常低价程序仅用于政府采购异常低价审查，不直接套用工程招投标低价或低于成本判断。

| 经验 | 产品动作 |
|---|---|
| 被认定异常低价后需要书面说明和证据，不是口头解释 | 生成异常低价证据包清单 |
| 财库〔2026〕2号触发线 | 低于平均报价 50%、低于次低报价 50%、低于最高限价 45%、评委专业判断过低时，应准备现场说明 | 触发线提示 |
| 财库〔2026〕2号触发线调整 | 采购人可结合项目提高前三项数值标准，但最高不超过 65%，以采购文件和审查记录为准 | 调整依据提示 |
| 说明时间 | 成本说明和证明材料提交时间一般不少于 30 分钟 | 现场说明提醒 |
| 证据可包括合同协议、营业执照、中标公示、中标通知书、付款证明、成本分析表、专项审计说明、原材料成本、人工成本、制造费用 | 成本合理性材料清单 |
| 核心是证明能履约，不是证明愿意亏 | 履约能力提示 |
| 审查归档 | 启动原因、审查意见、审查结果、供应商说明、证明材料和必要查询记录应归档 | 审查记录检查 |

### 5.6 电子监管

目标：提示电子投标过程中可能触发同源、关联或监管关注的合规风险。

| 风险痕迹 | 产品口径 |
|---|---|
| 制作机器码、网卡 MAC、硬盘序列号、文件创建标识码 | 合规独立制作提醒 |

| 报价规律异常、保证金路径异常 | 监管关注信号 |
| 关联公司同投、股东人员交叉、历史陪标关系 | 关联投标风险 |
| 异标段 IP/联系人雷同、节资率异常、异常高价投标、集中解密失败 | 监管关注信号 |
| 同一人员、同一模板或同一服务商代做同项目多家公司文件 | 文件同源和围串标疑点 |

## 6. 规则救济与履约结算层

规则救济与履约结算层是独立扩展层，重点不是“怎么写标书”，而是识别规则救济窗口、合同履约约束和工程结算审计风险。该层只输出风险识别、材料准备和程序提醒，不输出违法、犯罪、监管或司法定性。

### 6.1 异议质疑投诉

目标：判断发现异常条款、评审错误或其他投标人异常行为后，是否存在可操作的澄清、异议、质疑或投诉路径。

质疑投诉有效性预审应先看主体、证据、流程和书面形式，再判断实体理由是否值得展开。

| 检查项 | 判断 | 产品动作 |
|---|---|---|
| 澄清、异议、质疑、投诉区分 | 澄清是问清楚，异议/质疑/投诉是要求纠正或处理，不能混用 | 救济类型识别 |
| 期限起算点 | 采购文件、采购过程、中标结果的救济期限可能不同 | 救济时钟链 |
| 主体资格 | 供应商、潜在投标人、利害关系人等主体不同，适用路径不同 | 主体适格检查 |
| 请求事项 | 要求澄清、修改文件、重新评审、处理违规投标等必须具体 | 请求事项结构化 |
| 证据链 | 不能只写怀疑，应指向具体条款、公告、截图、公开信息、合同业绩矛盾或信用信息 | 证据链清单 |
| 证据具体性 | 证据必须指向具体违法违规点、具体主体、具体条款或具体事实 | 证据具体性检查 |
| 前置步骤 | 应先质疑/异议而直接投诉的，可能因程序缺失被驳回 | 前置程序检查 |
| 书面形式 | 质疑/异议材料应列明项目情况、事实理由、证据、联系方式、明确诉求 | 书面形式检查 |
| 签字盖章 | 法人签字、授权代表签字、单位盖章和授权材料要匹配适用程序 | 形式完整性检查 |
| 暂停效果差异 | 政府采购质疑期间不当然暂停；招标投标异议答复前通常影响继续推进 | 暂停效果提示 |
| 不予受理 vs 不成立 | 不予受理通常是程序问题，不成立是实体理由不足 | 程序/实体结果区分 |
| 非中标候选人异常 | 其他投标人的违法违规线索也可能影响竞争秩序，不只盯第一候选人 | 竞争异常线索 |
| 招标文件歧义或前后矛盾 | 资格、评分、格式、合同条款冲突时，应尽早澄清或异议 | 文件矛盾预警 |

救济时钟应结构化为规则码，避免只提示“尽快处理”：

| 场景 | 期限规则 | 产品动作 |
|---|---|---|
| 政府采购质疑 | 知道或应知权益受损之日起 7 个工作日内 | `GOV_PROC_CHALLENGE_7WD` |
| 政府采购质疑答复 | 采购人或代理机构收到质疑后 7 个工作日内答复 | `GOV_PROC_REPLY_7WD` |
| 政府采购投诉 | 答复期满后 15 个工作日内 | `GOV_PROC_COMPLAINT_15WD` |
| 财政部门投诉处理 | 一般自收到投诉后 30 个工作日内处理 | `GOV_PROC_COMPLAINT_HANDLE_30WD` |
| 资格预审文件异议 | 应在提交资格预审申请文件截止时间 2 日前提出 | `TENDER_PREQUAL_OBJECTION_2D` |
| 招标文件异议 | 应在投标截止时间 10 日前提出 | `TENDER_FILE_OBJECTION_10D` |
| 中标候选人公示 | 公示期不得少于 3 日 | `CANDIDATE_PUBLICITY_MIN_3D` |
| 工程招投标投诉 | 通常自知道或应知之日起 10 日内提出 | `TENDER_COMPLAINT_10D` |

合规边界：

- 串标、造假、骗取中标等只能作为合规风险线索或救济证据方向，不做行政、刑事或司法定性。
- 录音、截图、公开信息等证据类型只能作为证据候选提示，不鼓励非法取证。
- 输出应落到“建议澄清/异议/质疑评估”“建议补证据链”“建议核对期限和主体”，不写“违法确定成立”。

可落地输出：

- `remedy_window_state`。
- `challenge_evidence_chain_state`。
- `remedy_clock_rule_code`。
- `remedy_deadline_date`。
- `precondition_objection_required`。
- `remedy_subject_standing_check`。
- `document_conflict_remedy_hint`。
- `remedy_standing_check`。
- `evidence_specificity_level`。
- `pre_complaint_step_done`。
- `written_objection_formality`。

### 6.2 资格条件合法性

目标：识别资格条件、实质性要求和评分因素是否与项目实质需要直接相关，是否存在过度限制竞争风险。

| 检查项 | 判断 | 产品动作 |
|---|---|---|
| 非必要证书作为资格条件 | 证书是否与采购对象、强制法规和履约必要性直接相关 | 资格合法性风险 |

| 厂家授权 | 作为资格、实质性要求或高分项时，可能形成排他授权或唯一授权 | 授权排他性检查 |
| ISO 三体系等认证 | 不能所有项目机械加分，应看项目关联性和履约必要性 | 认证关联性检查 |
| CMA、检测资质、行业证书 | 是否可作为资格或评分，应看采购对象、法定强制要求和服务必要性 | 资质必要性检查 |
| 现场踏勘 | 一般不宜作为资格条件、实质性条件或加分条件，除非有明确合规依据 | 踏勘门槛检查 |
| 实质性要求标识 | 废标依据应看是否明确列入资格、符合性、星号、不可偏离或无效响应条款 | 实质性要求核验 |

- `qualification_legality_risk_hits`。
- `substantive_requirement_map`。
- `manufacturer_authorization_risk_check`。
- `certification_relevance_check`。

### 6.3 中标后合同与程序风险

目标：识别中标后放弃、递补、合同签订、履约保证和合同变更风险，避免把投标阶段胜率和中标后责任割裂。

| 检查项 | 判断 | 产品动作 |
|---|---|---|
| 放弃投标、弃标、放弃中标 | 开标前撤回、评标中放弃、候选人放弃、中标后不签合同，后果不同 | 放弃节点风险 |
| 无正当理由放弃中标 | 可能影响投标保证金、处罚或信用记录 | 保证金/信用风险 |
| 招标人临时加码 | 采购文件没有的额外条件、实质性变更合同或强加义务，应留证据并评估救济 | 合同加码风险 |
| 中标人不签合同 | 第二名不一定自动递补，可能递补、重新招标或监管处理 | 递补程序检查 |
| 政府采购合同核心条款 | 采购标的、数量、价格、核心服务内容、履约期限等不能随意变更 | 合同实质变更检查 |
| 履约保证金 | 采购文件未写，签约时临时要求新增或提高比例，存在争议空间 | 履约保证金检查 |
| 履约保函形式 | 是否必须银行保函，要看采购文件和合同是否限定 | 保函形式检查 |
| 总公司中标后分公司履约 | 需满足招标文件、合同约定、内部授权和实际履约责任，不能变成违法转包 | 履约主体检查 |
| 交付后新增授权条件 | 原厂授权、本地经销商授权等若未在采购文件和合同中明确，后置作为验收付款条件存在风险 | 验收授权卡点 |
| 验收额外条件 | 已交付后新增验收门槛、额外授权或本地服务商确认，可能造成付款受阻 | 验收加码风险 |
| 后置授权付款障碍 | 授权条件被用来替代技术商务验收，需核查采购文件、合同和验收标准 | 付款阻断检查 |

可落地输出：

- `post_award_contract_risk_hits`。
- `contract_change_substantiveness_check`。
- `award_abandonment_risk_state`。
- `second_ranked_supplier_procedure_state`。
- `post_award_authorization_gate`。
- `acceptance_extra_condition_risk`。

### 6.4 工程结算审计风险

目标：从工程履约和结算审计视角反推投标阶段报价、承诺和资料留存风险。

| 检查项 | 判断 | 产品动作 |
|---|---|---|
| 不平衡报价 | 高价清单项、低价清单项遇到工程量变化，结算时可能被审计调整或扣减 | 不平衡报价风险 |
| 固定单价合同 | 固定单价仍受合同条款、清单偏差、工程量变化和审计口径影响 | 固定单价结算风险 |
| 过程资料完整性 | 结算高度依赖标准施工段、隐蔽工程影像、验收记录、签证变更、材料进场、会议纪要 | 过程资料清单 |
| 后补资料 | 后补资料容易出现时间、签章、影像、逻辑和现场事实冲突 | 后补资料风险 |
| P 图或伪造资料 | 不只是结算被砍，还可能影响诚信、合同责任和法律风险 | 资料真实性红线 |
| 投标承诺落地 | 投标阶段承诺的人员、材料、工期、服务和技术路线，履约阶段要能被资料证明 | 承诺履约闭环 |

可落地输出：

- `settlement_audit_risk_hits`。
- `process_evidence_completeness_state`。
- `unbalanced_pricing_settlement_risk`。
- `commitment_delivery_trace_check`。

### 6.5 履约付款救济

目标：从中小企业款项支付、保证金、付款条件和审计结算依据角度，识别中标后现金流和救济风险。

| 检查项 | 判断 | 产品动作 |
|---|---|---|
| 付款期限 | 机关事业单位对中小企业付款一般 30 日，最长 60 日；大型企业对中小企业付款一般 60 日 | 付款期限风险 |
| 第三方付款条件 | 不得把收到第三方付款作为向中小企业付款的条件 | 第三方付款条件风险 |
| 非现金支付方式 | 不得强制商业汇票等非现金支付方式 | 非现金支付风险 |
| 审计作为结算依据 | 无明确法律、行政法规依据或合同约定，不宜强制以审计结果作为结算依据 | 审计结算依据风险 |
| 工程保证金 | 不得乱收保证金；不得限定只能现金，应接受金融机构保函等替代形式 | 保证金形式风险 |
| 拖欠款项 | 付款拖延、变相延期、付款条件加码时，应保留合同、验收、发票、付款节点和催告证据 | 欠款救济线索 |

可落地输出：

- `payment_term_violation`。
- `third_party_payment_condition_risk`。
- `audit_as_settlement_basis_risk`。
- `guarantee_cash_only_risk`。
- `arrears_complaint_path`。

### 6.6 专项整治与举报监督信号

目标：识别围标串标、买标卖标、黄牛掮客、长期固定供应商、参数定制和招标代理异常等可能进入专项整治、举报监督或倒查范围的线索。该能力只输出合规风险线索、留痕提醒和人工复核建议，不做行政、刑事或司法定性。

| 信号 | 判断 | 产品动作 |
|---|---|---|
| 投标人与投标人串通报价 | 报价规律、文件相似、IP/设备/人员关系等可能形成线索 | 串通投标线索 |
| 投标人与招标人串通 | 参数定制、异常沟通、提前锁定、非公开信息不对称 | 招投标串通线索 |
| 陪标、围标、轮流中标 | 长期陪跑、轮流中标、报价陪衬、固定供应商异常中标 | 竞争异常复盘 |
| 买标卖标、黄牛掮客 | 第三方掮客、异常中介费、买卖资格或项目机会线索 | 掮客风险线索 |
| 参数定制 | 技术参数、检测报告、厂家资料高度贴合单一供应商 | 控标线索联动 |
| 招标代理异常 | 代理行为通报、异常澄清、异常流程安排、投诉集中 | 代理风险提示 |
| 专项整治地区 | 地方专项治理、机器管招投标、举报奖励等公开政策 | 整治态势标记 |
| 举报奖励政策 | 有奖举报或举报专区可作为监督态势线索，需回到官方公告核验 | 举报监督信号 |

部分地区存在有奖举报政策，具体金额、适用范围和受理部门以官方公告为准；正式引用应优先回到省发改委、公共资源交易平台或正式公告。江西、河北、内蒙古、湖南等被提及的专项整治地区，应先标为“经验观察/待官方核验”，不能仅凭非正式经验材料写成正式高风险地区。

可落地输出：

- `bid_collusion_signal_type`。
- `bid_broker_signal`。
- `long_term_incumbent_signal`。
- `parameter_tailoring_signal`。
- `bid_traceability_risk`。
- `whistleblower_reward_policy_signal`。

## 7. 与 L0/D 文档关系

按当前 L0、D2、D3、D13 原口径，本经验库不能直接视为现有正式能力清单。它更准确的定位是：部分可作为现有公开证据主线的规则细化，更多是新增能力候选。

### 7.1 已有主线可承接的部分

这些内容与 L0/D13 当前“公开市场机会发现 + 证据包商业化”主线相容，可以作为现有能力的候选细化，但进入正式规则前仍要补字段血缘、证据等级和双闸门。

| 经验库内容 | 当前可承接面 | 说明 |
|---|---|---|
| 两法体系分流 | `procurement_regime`、`procurement_category`、`legal_system_type` 候选 | 可作为所有规则判断前置分类；`legal_system_type` 不直接替代正式对象字段 |
| 采购意向、招标计划、招标文件提前公示 | 公开市场机会发现、`notice_version_chain`、`clock_chain_profile` | 可提前形成关注池和准备窗口 |
| 项目来源库、重大项目清单、审批/土地/新闻等早期线索 | `public_chain`、`notice_version_chain`、source/route 候选 | 只作为公告前机会线索，需公开回链后才能进入正式证据链 |
| 限制性资格条款、参数定向、明显不当门槛 | `QUAL-001`、D13 A-05、`public_attack_surface` | 属于现有“限制性资格/条款性打击点”方向 |
| 公平竞争审查、本地保护、所有制/区域/产地差别待遇 | `QUAL-001`、D13 A-05、投诉公告复盘样本 | 可作为限制竞争和政策合规线索候选 |
| 地区规则差异画像 | 地方公共资源平台规则、历史项目样本、投诉公告复盘 | 可作为地区侧重点和工作量预估，不得直接写成地区好投/难投 |
| 采购需求管理、需求调查、需求量化 | `public_attack_surface`、需求侧规则候选 | 可作为控标源头识别，但需补需求形成证据 |
| 资格条件合法性、厂家授权、ISO/CMA、现场踏勘限制 | `QUAL-001`、D13 A-05、投诉公告复盘样本 | 可作为控标预测和限制竞争线索候选 |
| 评分办法异常、主观分异常、评分细项异常 | `SCORE-001`、D13 C-02 | 只在评分颗粒度公开充分时成立 |
| 资格条件评分、评分项未量化、同品牌/核心产品规则 | `SCORE-001`、D13 C-02、政府采购规则候选 | 可增强自评分和有效竞争家数测算 |
| AI 招投标政策、公开平台规则、招标文件检测场景 | `public_attack_surface`、政策/规则线索候选 | 只承接官方政策、公开平台规则和公开招标文件检测场景；投标文件内审不进入公开主链 |
| AI 项目信息采集、分类和项目档案 | Stage1/Stage7 内部经营评分扩展、source quality feedback | 可用于内部情报整理，不替代项目判断或正式证据核验 |
| 盲投采购方式分层、团队漏斗 | Stage7 内部经营评分扩展 | 可用于内部项目筛选，不等于对外中标承诺 |
| 评定分离、不排序候选、候选关系判断 | L0 候选推荐模式、evaluation coverage seed | 已有制度识别基础，但定标因素深挖仍需扩展 |
| 异常低价、严重不平衡报价、财库〔2026〕2号成本证据包 | L0 B/C 交界能力 | 只能在公开报价、评标办法和低价审查材料充分时形成线索 |
| 招标文件、答疑、补遗、版本链、时钟链 | `notice_version_chain`、`clock_chain_profile`、`field_lineage_record` | 属于现有公开链和版本/时钟主线 |
| 复盘样本：中标、废标、投诉、流标公告 | evaluation corpus / golden samples | 可用于训练和回归，不等于实时产品能力 |
| 异议、质疑、投诉公开结果 | complaint/evaluation corpus | 可用于规则校准和样本沉淀，不等于代客户发起救济 |
| 专项整治、举报监督、倒查风险 | 投诉公告、处罚案例、官方整治公告、公共资源平台通报 | 可作为合规态势线索，不做违法犯罪定性 |

### 7.2 新增能力候选

这些内容在当前 L0/D2/D3/D13 中没有完整正式对象、规则码、字段策略或客户交付边界。如果要产品化，必须作为新增能力走设计和治理。

| 新增能力候选 | 为什么是新增 | 可能落点 |
|---|---|---|
| 两法体系分流器 | 政府采购和招标投标规则路径不同，影响采购方式、监管和救济时钟 | 新增 legal-system classifier |
| 公告前机会层 | 当前主线主要从公告后开始，采购意向和招标计划属于更早阶段 | 新增 pre-notice opportunity watchlist |
| 公平竞争审查层 | 涉及正式审查规则和市场壁垒类型，不只是普通资格匹配 | 新增 fair-competition checker |
| 需求管理控标层 | 需要识别需求调查、需求量化、采购需求形成过程 | 新增 demand-management risk checker |
| 政府采购本国产品政策层 | 涉及 2026 后本国产品价格扣除和声明责任，需按政府采购产品场景触发 | 新增 domestic-product policy checker |
| 远程异地评标规则层 | 涉及适用范围、主副场责任、专家库共享、系统故障和资料保存 | 新增 remote-evaluation organization checker |
| 地区规则差异与地方画像层 | 需要地区历史样本、地方规则和官方平台政策，不是单项目事实 | 新增 regional-rule profile checker |
| AI评标与机器审查适配 | 涉及 AI 检测、投标合规自查、智能辅助评标、围串标识别和机器可读性 | 新增 ai-review readiness checker；投标文件 AI 自查、结构化响应、机器相似性风险需作为 bid-document review 扩展能力治理 |
| 项目来源库、AI采集与项目档案 | 需要跨来源采集、分类、项目档案和跟进记录，不是单条公告事实 | 新增 project-intelligence folder / source intelligence checker |
| 项目筛选 | 当前主线是发现可售机会，不是替投标人做投/不投决策 | 新增 `bid_selection_profile` 或作为 Stage7 内部经营评分扩展 |
| 自评分 | 需要消费“我方材料库/资质/业绩/人员”，当前对象体系主要围绕公开项目和竞争者 | 新增 supplier-side profile，或受控内部输入 |
| 盲投运营漏斗 | 需要团队产能、分工、反馈策略等内部经营信息 | 新增 blind-bid pipeline checker |
| 废标红线 | 授权、签章、投标函、声明函、税率、审计报告等不是当前主线正式规则 | 新增 bid-document review 能力 |
| 提交与开标流程 | 需要消费 CA、上传、加密、解密、二次报价、线下开标材料等过程信息 | 新增 submission/opening checklist |
| 暗标 | 字体、字号、符号、页眉页脚、元数据属于投标文件内容审查 | 新增 document QA/OCR/metadata 检查能力 |
| 正偏离 | 属于标书响应优化，不是公开证据线索发现 | 新增 response-quality assistant |
| 授权签章 | 属于投标文件合规审查 | 新增 bid-compliance checklist |
| 声明函 | 涉及政府采购/政策适用和投标材料真实性 | 新增 declaration-form checker，且需制度分型 |
| 税率审计 | 需要企业内部财务材料，不是公开项目材料 | 新增 supplier internal material QA |

| 电子标制作环境隔离 | 需要投标制作电脑、网络、CA、清单软件、打印扫描设备等内部过程信息 | 新增 bid-environment isolation checklist |
| 异议质疑投诉 | 需要制度时钟、主体适格、请求事项、证据链和程序结果语义 | 新增 remedy/challenge assistant |
| 资格条件合法性 | 涉及规则救济和竞争限制判断，不只是普通资格匹配 | 新增 qualification legality checker |
| 中标后合同与程序风险 | 涉及中标后合同、递补、放弃中标、履约保证等，不属于投前公开机会事实 | 新增 post-award risk checker |
| 中标后验收授权卡点 | 需要采购文件、合同、验收标准和付款沟通材料 | 新增 acceptance authorization gate checker |
| 工程结算审计风险 | 需要合同、清单、过程资料、签证变更等履约材料 | 新增 settlement/audit risk checker |
| 履约付款救济 | 涉及付款期限、第三方付款条件、审计结算依据、保证金形式和欠款救济 | 新增 payment-remedy checker |
| 专项整治与举报监督信号 | 涉及地方整治公告、举报奖励、倒查风险和线索归档 | 新增 enforcement-campaign / whistleblower-signal checker |

### 7.3 暂不应进入当前主线的部分

以下内容即使有业务价值，也不应在不改 L0/D13 的情况下混入现有正式主链：

- 代替投标人生成完整投标文件或承诺“提高中标率”的客户能力。
- 基于客户内部材料直接生成正式对外结论。
- 把暗标版式、签章、声明函、税率、审计报告等内部投标文件问题写成公开市场机会事实。
- 把电子监管特征码经验转成规避监管、规避同源识别或规避串标筛查方案。
- 未补政府采购质疑/投诉制度时钟时，直接把政府采购材料审查能力并入建设工程招投标主线。
- 把串标刑责、行政处罚或合同争议做成自动定性器。
- 把异议、质疑、投诉能力包装成代替律师、监管或司法判断的结论服务。
- 把招标投标法修订草案、政策趋势或 AI 生成提示当作现行规则直接判定。
- 把评审人员异常接触与利益输送线索写成犯罪成立或刑事结论。
- 把具体地区非正式经验材料写成地区确定性规则。
- 把圈子信息、社群消息或评论区内容直接写入正式证据链。

### 7.4 如果要产品化完整能力

如果要开发完整能力，至少需要补：

- D13 能力边界；
- D2 正式对象和字段；
- D3 规则码和结果语义；
- D6/D7 字段可见性和交付边界；
- D11 金标样本和回归测试；
- 对外表达和销售承诺限制。

## 10. 风险表达模板

| 场景 | 推荐表达 | 避免表达 |
|---|---|---|
| 两法分流 | 该项目应先区分政府采购法体系或招标投标法体系，再套用采购方式、监管和救济规则 | 所有项目都按同一套招投标规则判断 |
| 公告前机会 | 该采购意向/招标计划可进入关注池，建议持续跟踪公告和文件公示 | 这个项目一定会招 |
| 公平竞争 | 该条款可能涉及本地保护或差别待遇，建议核查公平竞争审查依据 | 这个条款一定违法 |
| 需求管理 | 该项目可能需要需求调查或需求量化支撑，建议核查需求形成依据 | 采购需求就是为某家写的 |
| 本国产品政策 | 该项目可能适用政府采购本国产品价格评审优惠，建议核查声明和成本占比 | 国产一定加 20 分 |
| 远程异地评标 | 该项目可能适用远程异地评标，建议核查主副场责任、系统和资料保存 | 熟人评标一定没用 |
| AI机器审查 | 该投标文件存在可读性、结构化或机器相似性风险，建议按评分点和证据链重整 | AI 一定会判串标 |
| 地区规则画像 | 该地区历史样本和平台规则显示存在某类规则侧重点，建议按本项目文件复核 | 这个地方一定好投/难投 |
| AI采集 | AI 可辅助按统一字段收集、清洗和分类项目信息，最终判断仍需人工和公开证据核验 | AI 会自动判断项目值不值得投 |
| 项目档案 | 该项目建议补齐基础信息、人员、竞争情报、时间节点和跟进记录后再判断投入 | 看到项目就马上做标书 |
| 盲投筛选 | 按采购方式、主观分占比、客观分可得性和价格分空间测算，当前盲投投入产出偏低 | 盲投一定烧钱 |
| 控标风险 | 疑似存在指向性/排他性条款，建议核查 | 这个标内定 |
| 参数风险 | 该参数组合可能显著缩小竞争范围 | 参数违法 |
| 评分风险 | 该评分项可能放大特定供应商优势 | 评委一定偏向某家 |
| 资料不对称 | 完成响应所需资料不足，建议澄清 | 招标人故意不给 |
| 废标风险 | 该项属于一票否决或高频废标风险 | 一定废标 |
| 胜率预测 | 按现有材料测算，当前得分空间偏低 | 一定不中 |
| 评审端风险 | 该项在评审端可能被认定为签署/格式/声明/暗标/报价风险，建议复核 | 专家一定废你 |
| 电子监管风险 | 该项存在同源痕迹或关联投标风险，建议合规整改 | 教你规避监管 |
| 制作环境隔离 | 该投标制作、上传或扫描环境存在同源痕迹，建议核查独立制作和留痕 | 换个设备规避监管 |
| 救济窗口 | 该事项可能存在澄清/异议/质疑评估价值，建议先核对期限、主体和证据链 | 这个肯定能投诉成功 |
| 救济有效性 | 该质疑/异议材料可能存在主体、证据、前置程序或书面形式缺口，建议补齐后再提交 | 只要有道理就一定受理 |
| 资格合法性 | 该资格条件与项目实质需要的关联性需要核查，可能存在限制竞争风险 | 资格条件违法确定成立 |
| 合同加码风险 | 该中标后新增要求可能涉及采购文件未列明的实质义务，建议留证并评估救济路径 | 甲方违法加码 |
| 验收授权卡点 | 该验收或付款要求可能属于交付后新增授权条件，建议核查采购文件、合同和验收标准 | 采购人恶意拒付 |
| 结算审计风险 | 该报价或承诺在结算审计阶段可能存在扣减或举证风险，建议补过程资料闭环 | 审计一定会砍 |
| 付款救济 | 该付款期限、第三方付款条件或保证金形式可能存在履约付款风险，建议核查合同和法定依据 | 甲方一定拖欠违法 |
| 评审人员异常接触 | 该线索可能涉及评审人员异常接触或利益输送风险，建议依法留证并走合规救济路径 | 评委犯罪成立 |
| 专项整治/举报监督 | 该地区或项目类型存在公开专项整治、举报监督或倒查风险信号，建议合规留痕并人工复核 | 这个项目一定会被查 |

## 11. 后续决策问题

下一步需要定九件事：

1. 两法体系分流器是否作为所有规则判断的前置分类器：`法律体系 + 资金来源 + 监管机构 + 采购/招标方式 + 救济路径`。
2. 公告前机会与政策规则层是否先做成独立关注池：`采购意向 + 招标计划 + 公平竞争 + 需求管理 + 本国产品政策 + 远程异地评标 + AI机器审查 + 地区规则画像`。
3. 主线能力层是否先做完整闭环：`项目筛选 + 控标预测 + 自评分 + 废标红线 + 盲投运营`。
4. 是否把项目来源库和 AI 采集先做成内部情报表：`来源渠道 + 生命周期阶段 + 字段模板 + 分类标签 + 质量反馈`。
5. 是否建立项目档案夹：`基础信息 + 人员信息 + 竞争情报 + 时间节点 + 跟进记录`。
6. 增强能力层是否先做成内部检查表：`暗标 + 正偏离 + 授权签章 + 声明函 + 税率审计 + 电子监管 + 制作环境隔离`。
7. 规则救济与履约结算层是否作为独立扩展能力推进：`异议质疑投诉 + 资格条件合法性 + 中标后合同与程序风险 + 工程结算审计风险 + 履约付款救济 + 验收授权卡点 + 专项整治与举报监督`。

## 12. 建议结论

如果不改变当前 L0/D13 产品方向，本草案只能作为：

1. 现有公开证据主线的规则候选池；
2. 复盘样本和风险词库；
3. 内部产品扩展蓝图。

如果要开发文档中的完整能力，则属于新增能力建设。

建议最小落地顺序：

1. 先做两法体系分流器：`法律体系 -> 资金来源 -> 监管机构 -> 采购/招标方式 -> 救济路径`。
2. 再做公告前机会与政策规则层：`采购意向/招标计划 -> 项目来源库 -> 地区规则画像 -> 公平竞争 -> 需求管理 -> 本国产品政策 -> 远程异地评标 -> AI机器审查`。
3. 再做 AI 情报采集和项目档案：`字段模板 -> 来源质量反馈 -> 分类标签 -> 项目档案夹 -> 时间节点链`。
4. 再做主线能力层：`项目筛选/盲投运营 -> 控标预测 -> 自评分/评委阅读成本 -> 废标红线`。
5. 再做增强能力层：`暗标 -> 正偏离 -> 授权签章 -> 声明函 -> 税率审计/异常低价证据包 -> 电子监管/制作环境隔离`。
6. 最后做规则救济与履约结算层：`异议质疑投诉有效性 -> 资格条件合法性 -> 中标后合同与验收授权 -> 工程结算审计风险 -> 履约付款救济 -> 专项整治与举报监督`。
