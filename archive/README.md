# 归档目录说明（最终收口模板）

## 1. 作用

本目录用于归档：

- 历史草稿
- 被 supersede 的版本
- 临时 round 包中不再作为正式仓库路径存在的中间交付物
- 迁移说明与废止说明

## 2. 归档原则

1. 归档不等于删除
2. 被归档文件不得继续作为正式权威引用
3. 正式引用面只能指向 README、状态板、裁决总表、现行 D1-D14、现行 contracts/handoff/testing/control 目录
4. 若文件已 superseded，必须在 archive 中保留来源、替代关系与生效区间

## 3. 目录建议

- `archive/docs/`
- `archive/contracts/`
- `archive/reports/`
- `archive/migration/`

## 4. 禁止事项

- 不得把 archive 中的文件继续当作现行母版
- 不得在 archive 中藏正式口径
- 不得只移动文件不补迁移说明
